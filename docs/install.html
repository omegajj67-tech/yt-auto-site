yt-auto 설치 가이드

설치 가이드

목표: 로컬 환경에서 yt-auto 실행(검증/렌더)까지 완료.

← 홈 사용법 spec 작성 코드

0) 준비물

필수: Node.js(권장 18+), FFmpeg, Git

  • Node 확인: node -v, npm -v
  • FFmpeg 확인: ffmpeg -version

1) FFmpeg 설치

yt-auto는 렌더 단계에서 FFmpeg를 호출합니다. 없으면 render가 실패합니다.

macOS(Homebrew)

brew update
brew install ffmpeg
ffmpeg -version

Ubuntu/Debian

sudo apt update
sudo apt install -y ffmpeg
ffmpeg -version

Windows(winget)

winget install Gyan.FFmpeg
ffmpeg -version

2) 코드 받기

git clone https://github.com/omegajj67-tech/legendary-spoon.git
cd legendary-spoon

yt-auto가 monorepo 하위 폴더로 들어있다면, 경로는 보통 yt-auto/ 입니다.

3) 의존성 설치 & 빌드

레포가 pnpm/yarn/npm 중 무엇을 쓰는지에 따라 명령이 달라집니다. 루트에 락파일을 보고 선택하세요.

  • pnpm-lock.yaml → pnpm
  • yarn.lock → yarn
  • package-lock.json → npm

pnpm

corepack enable
pnpm -v
pnpm install
pnpm -r build

npm

npm install
npm run build

yarn

yarn install
yarn build

4) 설치 확인(가장 먼저 할 것)

CLI 도움말이 떠야 정상입니다. (dist 경로는 빌드 결과에 따라 다를 수 있습니다.)

# 예시: cli.js 위치가 아래라고 가정
node yt-auto/packages/core/dist/cli.js --help

만약 dist/cli.js를 못 찾으면, 아래로 실제 파일 위치를 확인하세요.

find yt-auto -maxdepth 6 -type f -name "cli.js" -o -name "cli.mjs" -o -name "index.js" | head

5) 흔한 오류

  • ffmpeg not found → FFmpeg 설치/경로 문제
  • Cannot find module dist/cli.js → 빌드 안 됨 또는 출력 경로 다름
  • pnpm not found → corepack enable 실행 후 재시도
yt-auto 사용법

사용법

목표: spec.json을 검증(validate)하고, 영상으로 렌더(render)하고, 산출물을 확인하는 흐름.

← 홈 설치 spec 작성 코드

0) CLI 위치 확인

아래 경로는 예시입니다. 먼저 --help로 실제 커맨드/옵션을 확인하세요.

node yt-auto/packages/core/dist/cli.js --help

도움말에 render / validate / init / info 같은 명령이 보이면 정상입니다.

1) init (프로젝트 틀 생성)

샘플 spec/meta 디렉터리 구조를 생성하는 명령이 있다면, 가장 먼저 실행합니다.

# 예시
node yt-auto/packages/core/dist/cli.js init ./my-project

init이 없다면, 레포 안의 examples/ 또는 샘플 json을 찾아 사용하세요.

find yt-auto -maxdepth 6 -type d -name "examples" -o -name "example" -o -name "samples"
find yt-auto -maxdepth 8 -type f -name "*.json" | head -n 50

2) validate (spec 검증)

렌더 전에 spec.json이 스키마를 통과하는지 확인합니다.

node yt-auto/packages/core/dist/cli.js validate ./my-project/spec.json
  • 실패하면 에러 메시지에 “어느 필드가 빠졌는지/형식이 틀렸는지”가 나옵니다.
  • 검증 통과 후에 render로 넘어가세요.

3) render (영상 생성)

FFmpeg를 통해 최종 MP4를 생성합니다. 출력 경로 옵션은 render --help로 확인하세요.

# 예시
node yt-auto/packages/core/dist/cli.js render ./my-project/spec.json --out ./out/video.mp4

렌더 중 멈추거나 실패하면 아래를 먼저 확인하세요.

  • ffmpeg -version 이 정상인지
  • 입력 assets(이미지/영상/오디오) 경로가 존재하는지
  • 권한: 출력 폴더(./out) 생성/쓰기 권한

4) info (파이프라인 정보/진단)

파이프라인 단계/환경 정보를 보여주는 명령이 있다면, 이걸로 문제를 좁힙니다.

# 예시
node yt-auto/packages/core/dist/cli.js info

5) 문제 발생 시 “최소 재현” 체크리스트

  • 샘플 spec으로 먼저 render 성공시키기
  • 내 spec은 장면(scene) 1개 + 이미지 1개 + 오디오 없음으로 축소
  • 경로는 절대경로 또는 프로젝트 상대경로로 단순화
  • 출력은 짧은 3~5초로 제한
yt-auto spec 작성

spec 작성

목표: 최소 spec.json을 만들고 validate를 통과한 뒤 render로 확장.

← 홈 설치 사용법 코드

0) 원칙

  • 최소 구성으로 먼저 성공: 장면 1개 → 레이어 1개 → 짧은 길이
  • 경로는 단순하게: 프로젝트 폴더 기준 상대경로 추천
  • validate 통과 후에만 확장(트랜지션/오디오/자막 등)

실제 스키마 필드명은 레포의 schema 코드/README를 기준으로 맞추세요. 여기 예제는 “개념 템플릿”입니다.

1) 예제 폴더 구조(권장)

my-project/
  assets/
    bg.jpg
    music.mp3
  spec.json
  meta.json
  out/

이 구조로 잡으면 “assets 경로/출력 경로”가 단순해집니다.

2) 최소 spec.json (개념 템플릿)

아래는 “필드가 이런 레벨로 존재한다”는 최소 템플릿입니다. 실제 필드는 레포 스키마에 맞춰 조정하세요.

{
  "version": 1,
  "output": {
    "width": 1920,
    "height": 1080,
    "fps": 30,
    "durationSec": 5
  },
  "scenes": [
    {
      "id": "s1",
      "durationSec": 5,
      "layers": [
        {
          "type": "image",
          "src": "assets/bg.jpg",
          "fit": "cover"
        }
      ]
    }
  ]
}

3) meta.json (선택)

썸네일/메타데이터 생성이 파이프라인에 있다면, 아래 형태로 분리해 관리할 수 있습니다.

{
  "title": "yt-auto 테스트 영상",
  "description": "spec.json 기반 렌더 테스트",
  "tags": ["yt-auto", "ffmpeg", "automation"]
}

4) validate → render 흐름

# spec 검증
node yt-auto/packages/core/dist/cli.js validate ./my-project/spec.json

# 렌더(출력 옵션은 render --help로 확인)
node yt-auto/packages/core/dist/cli.js render ./my-project/spec.json --out ./my-project/out/video.mp4

5) 확장 체크리스트

  • 자막/텍스트 레이어 추가
  • 음악/나레이션 오디오 트랙 추가
  • 장면 2개 이상 + 트랜지션
  • 각 장면별 타임라인/레이어 순서 고정

확장할 때는 “한 번에 1개 기능만” 추가하고 매번 validate/render로 확인하세요.

6) 다음 작업(추천)

주인님이 원하시면, 레인이 현재 레포의 실제 스키마(필드명)에 맞춰 spec 예제를 “실제 동작하는 JSON”으로 재작성해드릴 수 있습니다.

그때는 yt-auto/packages/core/src/schema/ 또는 schema 관련 파일 경로 캡처/링크만 주시면 됩니다.
